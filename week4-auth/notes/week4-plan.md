# Week 4 计划 · 认证与鉴权（7/13–7/17）

> 本周有效学习时间为 D2–D5 四天。主线优先级：自建账号认证闭环 > RBAC 与关键测试 > OAuth2 流程 > W3 遗留实验。代码继续竖切进 `week2-express/src/`，本目录只保存计划和学习笔记。

## 本周验收闭环

```text
注册
→ 密码哈希入库
→ 登录校验密码
→ JWT 签发
→ 鉴权中间件验证 JWT
→ 受保护接口
→ 最小 RBAC（401 / 403）
→ 认证关键路径测试
```

掌握标准：能脱离代码讲清密码与 token 分别解决什么问题、成功与失败路径如何流经各层，并在周验收前从空白重建核心链路。

## 每日止步条件

### D2

- 数据库不出现明文密码。
- 注册响应不包含 password / passwordHash。
- 重复邮箱仍保持明确的冲突语义。
- 本人能解释为什么验证密码要 compare，而不是重新 hash 后比较字符串。

### D3

- 登录成功和失败路径都可演示。
- token 有明确 payload、签名密钥来源和过期时间。
- 中间件只负责认证，不把具体业务权限硬编码进去。

### D4

- 无身份返回 401；身份有效但权限不足返回 403。
- 客户端不能通过请求体自行提升角色。
- 至少 1–2 个认证关键测试通过。

### D5

- 主线 demo 可从注册走到受保护接口。
- OAuth2 掌握以能讲清流程和威胁点为最低标准。
- 周复盘明确一个已掌握点和一个仍需验证的问题。
