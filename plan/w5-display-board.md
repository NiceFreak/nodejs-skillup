### 0.1 展示 / 复习双视角（产品边界）

- 两种视角：**展示（demo，默认）** 面向外部观众，只呈现中性技术说明；**复习（review）** 面向本人，
  额外展开 W3 的「仍在路上」开放问题与自我复盘。
- 视角状态落在 **URL hash**（`?mode=review`），不再存 `localStorage`：干净链接永远是展示视角，
  上一次复习不会悄悄残留进下一次演示。
- 开关**直接放在展板上**（不在登录页）：当前视角一眼可见、可随手切换；进入复习视角时给醒目横幅，
  提示对外演示前切回展示。
- **待复核结论的口径提示不受视角影响**，展示视角也照常显示——对外不会只看到过度确定的版本。
- tab 与当前专题同样写进 URL hash（`tab=` / `topic=`），刷新保留、可直接链接到某个知识点。

## 1. 目的

- 本周（W5，Node.js 底层）是核心保护周，但随时可能被面试等打断。
- 对策：**能学多少学多少，学到的都用带 UI 的方式可展示出来**。做展示时相当于一次复盘，partial 进度也不等于零。
- 载体：`week8-fullstack/src/frontend/`（AGENTS.md 白名单，AI 可维护）。

## 2. 边界（红线）

- ✅ AI 可做：把**本人已跑出的现象、数值、判断表**用 UI 呈现。
- ❌ AI 不做：事件循环 / 阻塞 / stream / worker 等**核心 demo 代码**（黑名单，援助上限 L2）。
- 分界：展示板画的是「本人产出的数据 + 本人推导的模型」，**不是**「输入任意代码→自动算执行顺序」的在线模拟器——后者等于替本人完成理解 demo。

## 3. 设计原则

1. **少字多象**：每主题以一个可视化的可观察现象为主角，文字降为旁注。
2. **真实数据驱动**：画的是本人实验测到的真实数值，不是装饰图形。
3. **粗颗粒**：每周 demo 仅约 10 分钟，做「几个能一眼看懂的大块」，不做细碎分步。

### 3.1 两个交付轴（2026-07-22 已拍板）

- **可视化复习轴**：按知识点组织，覆盖已经完成并验收的学习成果；目标是帮助本人回看、比较和重新推导，不受 demo 时长限制。
- **Demo 展示轴**：从复习板中挑少数知识点精讲，单独编写讲稿；不要求页面结构等于演讲顺序，也不要求展示全部学习内容。
- 两者解耦：复习板追求知识结构完整，讲稿追求有限时间内的重点与叙事。
- 页面弱化 D1 / D2 等日程标签，统一改用「知识点 1 / 2…」；未完成主题不提前占据主视图。
- **切分主轴是「知识点递进 / 相关」，不是「哪天学了什么」**（2026-07-23 明确）：复习按概念组织，
  日程只留在笔记来源里。已验收但非本机实测的「链路理解 / 判断模型」（如「I/O 慢」）也作为知识点纳入，
  但必须用来源口径标清「非实测」，与实测知识点区分。

## 4. 展示板结构

在现有前端加一个与报表看板并列的视图。顶部用知识点导航切换，中间只呈现当前知识点的专属可视化，底部收口运行时判断与业务映射：

| 字段 | 内容 |
|---|---|
| 状态 | 已完成 / 进行中 / 未开始 |
| 可观察证据 | 本人实验的真实输出（可视化主角） |
| 运行时判断 | 能迁移回业务的一句话结论 |
| 映射回 Week2–4 | 对应真实后端代码（bcrypt=CPU、mongo 查询=I/O…） |

未学到的主题不放大卡占位；完成并验收后再进入知识点导航，避免计划信息压过真实产出。

## 5. 各主题视觉呈现（定调，非最终）

W5 运行时板现有 **4 个知识点**，按运行时判断递进（调度边界 → CPU 阻塞 → 线程池排队 → 外部 I/O 等待）；
底部「三类慢判断表」把后三者收成一张分诊表，每张卡片可跳回对应知识点：

- **知识点 1 · 事件循环调度边界**（已完成，实测）：调用栈、nextTick、microtask、libuv 阶段四层模型呈现推导顺序；顶层与 I/O callback 场景作为对照。
- **知识点 2 · 同步 CPU 阻塞**（已完成，实测）：真实时间线对照 `20ms→late 0` 与 `2000ms→late 1904`，标出 CPU 占用、timer 到期和 callback 执行位置。
- **知识点 3 · 线程池排队与 UV_THREADPOOL_SIZE**（已完成，实测）：pbkdf2×8 在 SIZE=4/8 下的分批 vs 聚批，worker/队列机制示意 + 排队回放。
- **知识点 4 · 外部 I/O 等待**（已完成，**链路理解 / 判断模型，非实测**）：分诊推理（观测→排除/指向→工作假设→证据边界）、等待远端 2s 内 OS/libuv/Node/V8 四层职责泳道、poll 等待 vs 同步 while 对照、fd 释义。来源口径已标「非本机实测」。
- **stream / 背压**（未开始）：producer/consumer 缓冲条动画，高水位触发 drain 变色；旁边 `readFile` vs stream 内存占用对比条。
- **错误边界 / worker**（未开始）：错误流向图、主线程阻塞 vs worker offload 响应能力对照——学到后回填。
